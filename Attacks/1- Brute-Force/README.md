# Attack 01 – SSH Brute Force

## Attack Summary

| Item | Value |
|------|-------|
| Attack Type | SSH Brute Force |
| Tool Used | Hydra |
| Attacker | Kali Linux |
| Victim | Windows 10 |
| SIEM | Wazuh |
| Detection Status | Detected |
| Result | True Positive |
| MITRE Technique | T1110 |

## Objective

The objective of this simulation was to generate multiple failed SSH authentication attempts against a Windows 10 system running OpenSSH and observe how Wazuh detects and logs the activity.

This exercise demonstrates how a SOC analyst can identify brute-force attacks using Windows Security Events and Wazuh alerts.

## Lab Environment

Attacker: Kali Linux

Victim: Windows 10

Monitoring:
- Wazuh Agent
- Sysmon

SIEM:
- Wazuh Manager

## Attack Execution

The attack was performed from the Kali Linux attacker machine against the Windows 10 victim running the OpenSSH service.

Hydra was used to automate SSH authentication attempts using a predefined username and a password wordlist. Multiple login attempts were generated within a short period to simulate a brute-force attack against the SSH service.

The objective was to generate authentication failures that could be detected and analyzed by Wazuh.

## Detection

During the attack, Wazuh successfully collected and analyzed the authentication events generated by the Windows 10 endpoint.

Multiple alerts related to failed SSH authentication attempts were observed in the Wazuh dashboard. These alerts contained information such as the event timestamp, source IP address, target system, event description, and severity level.

The repeated authentication failures indicated behavior consistent with a brute-force attack and demonstrated Wazuh's ability to detect suspicious login activity.

## Analysis

Analysis of the collected logs showed a high number of failed authentication attempts originating from a single source IP address within a short period.

This pattern is characteristic of a brute-force attack, where an attacker repeatedly attempts different passwords until successful authentication is achieved.

Correlation of the authentication events and Wazuh alerts confirmed that the activity originated from the Kali Linux attacker machine used in the controlled laboratory environment.

## Analyst Decision

Classification: True Positive

Severity: Medium

Reason:

The alerts were generated due to multiple failed SSH authentication attempts originating from a single source IP address within a short time period. Investigation confirmed that the activity was part of a controlled brute-force attack.

Recommended Response:

- Verify the legitimacy of the source IP.
- Review affected user accounts.
- Check for any successful logins following the failed attempts.
- Block or rate-limit the attacking IP if unauthorized.
- Monitor for additional suspicious authentication activity.

## Impact Assessment

No successful authentication occurred during the simulation.

The attack demonstrated how repeated login attempts can indicate an attempt to compromise credentials.

If successful, an attacker could gain unauthorized access to the Windows system through SSH.

## Indicators of Compromise

- Source IP: 10.0.3.9
- Destination IP: 10.0.3.11
- Protocol: SSH (TCP/22)
- Multiple failed SSH login attempts
- 73 authentication failures
- High frequency of login attempts
- Wazuh authentication alerts


## Mitigation

The following security measures are recommended to reduce the risk of brute-force attacks:

- Enforce strong password policies.
- Implement account lockout policies after repeated failed login attempts.
- Enable Multi-Factor Authentication (MFA) where possible.
- Restrict SSH access to trusted IP addresses.
- Continuously monitor authentication logs for suspicious activity.
- Configure alerting for excessive failed login attempts.

## Lessons Learned

This exercise demonstrated how repeated authentication attempts can be detected through centralized log monitoring.

It also highlighted the importance of correlating authentication events with SIEM alerts to quickly identify credential-based attacks and support timely incident response.

## Conclusion

The simulated SSH brute-force attack successfully generated authentication events that were collected by the Wazuh Agent and analyzed by the Wazuh Manager.

The investigation confirmed that the alerts accurately represented brute-force activity, demonstrating the effectiveness of centralized log monitoring for detecting credential-based attacks in a controlled SOC environment.
